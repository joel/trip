 Step-by-Step Implementation

 Step 1: Add Gems to Gemfile

 Add after the authorization section (line ~42):
 # Business logic
 gem "dry-monads", "~> 1.6"

 # Export
 gem "gepub"

 Run bundle install to update Gemfile.lock.

 Step 2: Update Roles Configuration

 Change config/initializers/roles.rb to:
 Rails.configuration.roles = %i[superadmin contributor viewer guest].freeze

 Critical: This changes bit positions!
 - Old: superadmin=0, admin=1, member=2, contributor=3, guest=4
 - New: superadmin=0, contributor=1, viewer=2, guest=3

 Existing users with roles_mask=16 (old guest at bit 4) will need migration.
 Create a data migration to fix existing roles_mask values.

 Step 3: Update Policies for Role Rename

 - app/policies/application_policy.rb: Rename admin? to superadmin? using only user&.role?(:superadmin)
 - app/policies/user_policy.rb: Update all methods to call superadmin?

 Step 4: Create BaseAction

 Create app/actions/base_action.rb:
 # frozen_string_literal: true

 class BaseAction
   include Dry::Monads[:result, :do]
 end

 Step 5: Install Action Text

 Run bin/rails action_text:install which will:
 - Create migration for action_text_rich_texts and active_storage_blobs/active_storage_attachments
 - Add JS entries for Action Text

 Then run bin/rails db:migrate.

 Step 6: Add SeaweedFS Config to storage.yml

 Add to config/storage.yml:
 seaweedfs:
   service: S3
   endpoint: <%= ENV.fetch("SEAWEEDFS_ENDPOINT", "http://localhost:8333") %>
   access_key_id: <%= ENV.fetch("SEAWEEDFS_ACCESS_KEY_ID", "any") %>
   secret_access_key: <%= ENV.fetch("SEAWEEDFS_SECRET_ACCESS_KEY", "any") %>
   bucket: <%= ENV.fetch("SEAWEEDFS_BUCKET", "catalyst") %>
   region: us-east-1
   force_path_style: true

 Step 7: Uncomment PWA Routes and Update Manifest

 In config/routes.rb, uncomment:
 get "manifest" => "rails/pwa#manifest", as: :pwa_manifest
 get "service-worker" => "rails/pwa#service_worker", as: :pwa_service_worker

 Update app/views/pwa/manifest.json.erb:
 - Name: "Trip Journal"
 - Short name: "TripJournal"
 - Theme color: match design tokens
 - Background color: match design tokens

 Add <link rel="manifest" href="/manifest.json"> to app/views/layouts/application_layout.rb head section.

 Step 8: Event Subscriber Skeleton

 Create config/initializers/event_subscribers.rb:
 # frozen_string_literal: true

 # Event subscriber registry for Rails.event structured events.
 # Subscribers will be registered here as domain features are implemented.
 #
 # Example:
 #   Rails.event.subscribe("trip.created", TripSubscriber)

 Step 9: Update Factory and Specs

 - Update spec/factories/users.rb: rename :admin trait to :superadmin, add :contributor and :viewer traits
 - Update spec/models/user_spec.rb: test new role set
 - Update any request/system specs referencing :admin trait

 Step 10: Data Migration for Roles

 Create migration to convert existing roles_mask values from old bit positions to new ones.

 Risks

 1. Roles bit position change - Existing users in DB will have wrong roles after changing the list. Mitigated by data migration.
 2. Action Text install - May add unexpected JS or CSS dependencies. Monitor importmap and asset pipeline.
 3. Lexxy - Deferred to later phase. Using Action Text with default Trix for now.
